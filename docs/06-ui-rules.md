# UI Rules

## Visual direction
The product should feel:
- boutique
- premium
- modern
- elegant
- calm
- refined

Avoid:
- flashy gradients everywhere
- cramped layouts
- overly cute or noisy visuals
- “cheap booking site” feeling
- excessive border clutter

## Public website rules
- strong hero with tasteful imagery
- clean typography hierarchy
- generous whitespace
- subtle hover states
- polished room cards
- clear CTAs
- mobile-first but desktop-refined
- room detail pages should feel immersive and editorial

## Admin portal rules
- dense but readable
- clear filters and table actions
- calendar/grid views where helpful
- floor/room management should prioritize clarity over decoration
- statuses must be easy to scan
- use badges consistently

## Design tokens (guideline level)
- neutral base palette
- one restrained brand accent
- rounded corners but not overly soft
- consistent spacing scale
- strong card and panel hierarchy
- accessible contrast

## Form rules
- keep public forms short
- inline validation
- show helpful empty states
- use progressive disclosure
- avoid making guest create password too early
- admin forms should support draft/save patterns where useful

## Room content pages should always prioritize
1. title
2. gallery
3. short value proposition
4. occupancy/bed/size facts
5. amenities
6. price display if enabled
7. request CTA

## Bilingual UI rules
- all user-facing UI must support VI/EN
- hospitality tone should feel natural, not machine-translated
- do not mix languages within one UI block unless explicitly needed
- fallback strategy must be defined if EN content is missing

## Status UX rules
Use consistent status colors and labels for:
- request states
- reservation states
- payment states
- room statuses
- publish states

## Loading / skeleton rules
- use skeletons on public listing pages
- avoid jarring layout shift
- keep perceived speed high

## Accessibility basics
- keyboard reachable controls
- alt text support for media
- labels for all inputs
- visible focus state
- responsive tap targets on mobile
